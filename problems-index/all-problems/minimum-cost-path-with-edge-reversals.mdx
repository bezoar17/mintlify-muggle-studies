      <p>You are given a directed, weighted graph with <code>n</code> nodes labeled from 0 to <code>n - 1</code>, and an array <code>edges</code> where <code>edges[i] = [u<sub>i</sub>, v<sub>i</sub>, w<sub>i</sub>]</code> represents a directed edge from node <code>u<sub>i</sub></code> to node <code>v<sub>i</sub></code> with cost <code>w<sub>i</sub></code>.</p>  <p>Each node <code>u<sub>i</sub></code> has a switch that can be used <strong>at most once</strong>: when you arrive at <code>u<sub>i</sub></code> and have not yet used its switch, you may activate it on one of its incoming edges <code>v<sub>i</sub> &rarr; u<sub>i</sub></code> reverse that edge to <code>u<sub>i</sub> &rarr; v<sub>i</sub></code> and <strong>immediately</strong> traverse it.</p>  <p>The reversal is only valid for that single move, and using a reversed edge costs <code>2 * w<sub>i</sub></code>.</p>  <p>Return the <strong>minimum</strong> total cost to travel from node 0 to node <code>n - 1</code>. If it is not possible, return -1.</p>   <p><strong class="example">Example 1:</strong></p>  <div class="example-block"> <p><strong>Input:</strong> <span class="example-io">n = 4, edges = [[0,1,3],[3,1,1],[2,3,4],[0,2,2]]</span></p>  <p><br/> <strong>Output:</strong> <span class="example-io">5</span></p>  <p><strong>Explanation: </strong></p>  <p><strong><img alt="" src="https://assets.leetcode.com/uploads/2025/05/07/e1drawio.png" style="width: 171px; height: 111px;" /></strong></p>  <ul>  <li>Use the path <code>0 &rarr; 1</code> (cost 3).</li>  <li>At node 1 reverse the original edge <code>3 &rarr; 1</code> into <code>1 &rarr; 3</code> and traverse it at cost <code>2 * 1 = 2</code>.</li>  <li>Total cost is <code>3 + 2 = 5</code>.</li> </ul> </div>  <p><strong class="example">Example 2:</strong></p>  <div class="example-block"> <p><strong>Input:</strong> <span class="example-io">n = 4, edges = [[0,2,1],[2,1,1],[1,3,1],[2,3,3]]</span></p>  <p><br/> <strong>Output:</strong> <span class="example-io">3</span></p>  <p><br/> <strong>Explanation:</strong></p>  <ul>  <li>No reversal is needed.   <Tab title="Solution" icon="lightbulb" iconType="regular" >
    <Card>
      Since we can use the reverse edge with double weight, just add that edge as an option. And now wrt to constraint of only 1 incoming edge being reversed. That is handled automatically by the fact that we are not going to visit a cell twice.

So, add all edges including reversed ones, and do a simple djikstra.
</Card>

```csharp title="Minimum Cost Path with Edge Reversals" lines
public class Solution 
{
    public int MinCost(int n, int[][] edges) 
    {
        var graph = new Dictionary<int, List<(int neighbor, int weight)>>();
        
        // Initialize graph
        for (int i = 0; i < n; i++) 
        {
            graph[i] = new List<(int, int)>();
        }
        
        // Build graph
        foreach (var edge in edges) 
        {
            int u = edge[0], v = edge[1], w = edge[2];
            graph[u].Add((v, w));
            graph[v].Add((u, 2 * w));
        }
        
        // Priority queue using SortedSet for min heap behavior
        var pq = new SortedSet<(int dist, int node)>();
        var visited = new HashSet<int>();
        
        pq.Add((0, 0));
        
        while (pq.Count > 0) 
        {
            var (dist, node) = pq.Min;
            pq.Remove(pq.Min);
            
            if (visited.Contains(node)) 
            {
                continue;
            }
            
            visited.Add(node);
            
            if (node == n - 1) 
            {
                return dist;
            }
            
            foreach (var (neighbor, w) in graph[node]) 
            {
                if (!visited.Contains(neighbor)) 
                {
                    pq.Add((dist + w, neighbor));
                }
            }
        }
        
        return -1;
    }
}
```
