    <Card>
      <p>Given an integer array nums, return all the triplets <code>[nums[i], nums[j], nums[k]]</code> such that <code>i != j</code>, <code>i != k</code>, and <code>j != k</code>, and <code>nums[i] + nums[j] + nums[k] == 0</code>.</p>  <p>Notice that the solution set must not contain duplicate triplets.</p>   <p><strong class="example">Example 1:</strong></p>   <strong>Input:</strong> nums = [-1,0,1,2,-1,-4] <br/> <strong>Output:</strong> [[-1,-1,2],[-1,0,1]] <br/> <strong>Explanation:</strong>  nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0. nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0. nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0. The distinct triplets are [-1,0,1] and [-1,-1,2]. Notice that the order of the output and the order of the triplets does not matter.   <p><strong class="example">Example 2:</strong></p>   <strong>Input:</strong> nums = [0,1,1] <br/> <strong>Output:</strong> [] <br/> <strong>Explanation:</strong> The only possible triplet does not sum up to 0.   <p><strong class="example">Example 3:</strong></p>   <strong>Input:</strong> nums = [0,0,0] <br/> <strong>Output:</strong> [[0,0,0]] <br/> <strong>Explanation:</strong> The only possible triplet sums up to 0.    <p><strong>Constraints:</strong></p>  <ul>  <li><code>3 &lt;= nums.length &lt;= 3000</code></li>  <li><code>-10<sup>5</sup> &lt;= nums[i] &lt;= 10<sup>5</sup></code></li> </ul>

    <Card>
      2 pointer approach.

Sort the array. 

Fix first no., sum to 0 in right side of this array using 2 pointers left and right.

if fixed no. &gt; 0, return

else, move pointers in the search space based on sum.

To avoid duplicates, increment whenever we see duplicate elements.
    </Card>
  <Expandable title="code">
    <CodeGroup dropdown>
```ruby title="3Sum" lines
# @param {Integer[]} nums
# @return {Integer[][]}
def three_sum(nums)
  answers = []
  size = nums.sort!.size
  nums[..size-3].each_with_index do |num, i|
    return answers if num > 0 # search to right will not yeild 0 sum
    next if i > 0 && num == nums[i - 1] # move to avoid duplicate and reach the next

    left, right = i + 1, size - 1
    while left < right
      sum = num + nums[left] + nums[right]
      if sum > 0
        right -= 1
      elsif sum < 0
        left += 1
      else
        left_num, right_num = nums[left], nums[right]
        answers << [num, left_num, right_num]
        
        left +=1 while left_num == nums[left] # move to avoid duplicate and reach the next
        right -= 1 while right_num == nums[right] # move to avoid duplicate and reach the next
      end
    end
  end
  answers
end
```
