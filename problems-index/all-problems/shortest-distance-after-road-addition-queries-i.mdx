    <Card>
      Editorial's solution is brilliant, going from the right to left, and calculating dist to n-1, is dist_i = min(all neighbor nodes) + 1; 

as all neighbor nodes are to the right, when going from R-&gt;L, we can store dist in array, and all neightbors will have been calculated, already. 

And for each query [L,R] only [0..L] region will be affected, so we recompute dist for these.
    </Card>

```ruby title="Shortest Distance After Road Addition Queries I" lines
# @param {Integer} n
# @param {Integer[][]} queries
# @return {Integer[]}
def shortest_distance_after_queries(n, queries)
  adjacency_h = Hash.new {|h,k| h[k]=[] }
  (0...(n-1)).each { |i| adjacency_h[i] << i+1 }
  
  distance = (0...n).to_a.reverse
  
  queries.map do |start_idx, end_idx|
    adjacency_h[start_idx]<<end_idx; # update link
    
    # compute new min, only for affected region, start_idx and below
    start_idx.downto(0).each do |node|
      distance[node] = 1 + adjacency_h[node].map { |neighbor| distance[neighbor] }.min
    end
  
    distance[0]
  end
end
```
