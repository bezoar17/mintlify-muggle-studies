  <Tab title="Solution" icon="lightbulb" iconType="regular" >
    <Card>
      First of all get the base_profit. This is the case where we dont use the modification of the subarray of length k.

If we were doing the modification, we have n-k+1 options of (l,r). For each modification, the change happening is removal of base_profit(l,r) and addition of all prices in second half of (l,r). Both these things are sub-array sums. 
So, if we precalculate the prefix for base_profit and prices, we can easily calculate the increase we can get from choosing a modification. We calculate max possible gain from a modification, and add that to the base profit. 
    </Card>

```ruby title="Best Time to Buy and Sell Stock using Strategy" lines
# @param {Integer[]} prices
# @param {Integer[]} strategy
# @param {Integer} k
# @return {Integer}
class PseudoSegTree
  def initialize(nums)
    @prefix_arr = [0]
    (0...nums.size).each {|x| @prefix_arr << @prefix_arr.last + nums[x] }
  end

  # [l,r)
  def query(l,r)
    @prefix_arr[r] - @prefix_arr[l]
  end
end

def max_profit(prices, strategy, k)
  n = prices.size
  base_profit = prices.zip(strategy).map {|action, price| action*price }

  prices_psg = PseudoSegTree.new(prices) # to get sum[l,r], do psg.query(l, r+1)
  base_profit_psg = PseudoSegTree.new(base_profit)

  max_gain = (0..(n-k)).map do |l|
    r = l + k - 1; second_half_start = l + k/2
    prices_psg.query(second_half_start, r+1) - base_profit_psg.query(l, r+1)
  end.max
  
  base_profit.sum + [0, max_gain].max
end
```
