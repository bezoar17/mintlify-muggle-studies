      <p>Given an <code>m x n</code> integers <code>matrix</code>, return <em>the length of the longest increasing path in </em><code>matrix</code>.</p>  <p>From each cell, you can either move in four directions: left, right, up, or down. Moving diagonally is not allowed.   <p><strong class="example">Example 3:</strong></p>   <strong>Input:</strong> matrix = [[1]] <br/> <strong>Output:</strong> 1    <p><strong>Constraints:</strong></p>  <ul>  <li><code>m == matrix.length</code></li>  <li><code>n == matrix[i].length</code></li>  <li><code>1 &lt;= m, n &lt;= 200</code></li>  <li><code>0 &lt;= matrix[i][j] &lt;= 2<sup>31</sup> - 1</code></li> </ul>

  <Tab title="Solution" icon="lightbulb" iconType="regular" >
    <Card>
      Simple dfs with memoization; max len = prev + 1 if value is greater, kinda like bellman ford, which calculates all possible variations. Just the caching improves the speed.
    </Card>
  <Expandable title="code">
    <CodeGroup dropdown>
```ruby title="Longest Increasing Path in a Matrix" lines
# @param {Integer[][]} matrix
# @return {Integer}
def longest_increasing_path(matrix)
  rows = matrix.length - 1
  columns = matrix[0].length - 1

  directions = ->(row, column) { [[1,0],[-1,0],[0,1],[0,-1]].map { |r, c| [r + row, c + column] } }
  in_bounds = ->(row, column) { row.between?(0, rows) && column.between?(0, columns) }
  directions_in_bounds = ->(row, column) { directions.call(row, column).select { |r, c| in_bounds.call(r, c) && matrix[row][column] < matrix[r][c] } }
  
  cache = {}
  
  dfs = ->(row, column) do
    cache[[row, column]] ||= directions_in_bounds.call(row, column).reduce(1) { |result, (r,c)| [result, dfs.call(r, c) + 1].max }
  end

  (0..rows).each { |row| (0..columns).each { |column| dfs.call(row, column) } }

  cache.values.max
end
```
