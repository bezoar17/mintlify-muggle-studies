      An element having 4 cells as neighbors, must have degree of atleast 4. If there is a node with 1 as degree, then row count is 1. You cannot have more than 1 row, and also cant have more than 1 node as degree 1. So, process nodes accordingly. Think of degrees 1,2,3,4 and where will these nodes go. 

There's 2 cases
Either you have a single row - so 2 numbers have exactly one neighbor
or you have more than one row, so 4 numbers have exactly 2 neighbors.

```ruby title="Construct 2D Grid Matching Graph Layout" lines
# @param {Integer} n
# @param {Integer[][]} edges
# @return {Integer[][]}
def construct_grid_layout(n, edges)
  graph = Hash.new {|h,k| h[k]=[]}
  edges.each {|u,v| graph[u]<<v; graph[v]<<u; }
  degrees = (0...n).map { |i| graph[i].size }
  root = degrees.each_with_index.min[1]
  
  visited = Array.new(n, false)
  visited[root] = true
  path = [root]

  # build first row of the grid
  (0...n).each do |i|
    node = path[i]
    break if path.size>=2 && degrees[path[-1]] == degrees[root] # first and last of first row have same degree
    graph[node].sort_by{ |i| degrees[i] }.each do |neighbor|
      if !visited[neighbor] && degrees[neighbor] <= degrees[root] + 1
        path << neighbor
        visited[neighbor] = true
        break
      end
    end
  end

  c = path.size; r = n/c
  grid = Array.new(r) { Array.new(c) }
  grid[0] = path

  # build other rows, based on previous row's cell
  (1...r).each do |i|
    (0...c).each do |j|
      graph[grid[i-1][j]].each do |neighbor|
        next if visited[neighbor]
        grid[i][j]=neighbor
        visited[neighbor]=true
        break
      end
    end
  end

  grid
end
```
